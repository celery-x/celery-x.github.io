java面经重点整理-2023-04

自动封箱

Integer i1 = new Integer(1);
Integer i2 = new Integer(1);

System.out.println(i1 == i2);

为false

Integer i1 = 1;
Integer i2 = 1;

System.out.println(i1 == i2);

为ture 

Integer i1 = 333;
Integer i2 = 333;

System.out.println(i1 == i2);

为false 

自动封箱时，若在-128，127之间，会使用同一对象。
即在第一次封箱某个数的时候，对其缓存，再次封箱这个数的时候，直接返回缓存的obj。
仅限于自动封箱，new 还是原本的逻辑。

子父类关系

子类的构造过程中必须调用父类的构造方法，如果没有显示super(xxx);会调用默认构造方法即super();

为什么String不能被继承

首先确定一点，String被final修饰，是无法被继承的。
核心原因其实就是java设计不想让String的实现细节被改变。
字符串类被当做基本类型用在各个地方，如果随意继承，那么很多地方都设计都会面临风险，比如说类加载器就用到了String作为参数

finally中的代码一定会执行

那么finally和return会有什么化学反应？

如果return时，假设return o；
会把o指向的地方准备返回。
然后执行finally，finally里只有更改原本o指向的地方都内容，才能更改返回值，比如o.i = 12;
如果是改变o的值，比如说o = new Object(),是不会改变返回值的。

如果finally中有return，那么以finally中的return为准。

字符串常量池

1.7以前，执行String.intern时，若常量池有该字符串，会返回该字符串引用，若没有，则会在常量池创建该字符串并返回引用；
1.7之后，如没有，会在常量池创建一个该字符串的引用（该字符串在堆中）

synchronized的升级机制

无锁，加锁代码还没被执行过
偏向锁，第一个执行的线程会被标记为偏向线程，该线程后续到来时，都可以直接拿到锁。其他线程来的时候,原线程若已不存在，或不再使用这个锁，会通过CAS尝试偏向自己。否则就会升级
轻量级锁，当其他线程要执行加锁代码时，发现偏向锁指的不是自己，就会使该锁升级为轻量级锁，即CAS尝试获取锁。
重量级锁，当自旋的次数达到10（默认情况），该锁就会升级为重量级锁，即未获取到锁的线程阻塞等待唤醒。

ThreadLocal的原理

所以ThreadLocal所在对象被回收了之后，线程中的map以来对其有引用，回收不掉，所以用的弱引用，这样，当ThreadLocal除了map外无处可以获取到时（你能拿到key，才能map.get(key)呀），就可以回收key
且在Thread中get set remove时被删除key为null的value

具体为，get在计算key的hash时，如果直接code的位置就是该key，则直接返回，否则说明发送了hash冲突，就从该位置开始，遍历数组（底层是Entry数组进行存储的）；set也是在散列时顺手清除
